# ADR-006 异步队列与 VRAM 互斥储备方案

## 状态

Proposed

## 决策

异步任务队列和 VRAM 互斥管理器仅作为储备方案记录，本轮不实现。

## 理由

- 当前 MVP 范围明确排除复杂调度系统
- 现阶段真实并发需求尚未成立
- 过早实现会显著抬高复杂度和维护成本

## 触发条件

- 浏览器接入后出现频繁超时
- 本地多图并发提交成为真实需求
- 引入本地 LLM 翻译导致 GPU 资源争抢

## 放弃方案

- 在 M1/M2 阶段直接引入队列
- 在当前单请求模型下先做显存调度层

## 复审时机

- Gate-M4 前后
- 任一触发条件被验证成立时
